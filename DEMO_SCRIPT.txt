[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let us walk through the architecture of how this works under the hood. 
First, when a user submits an emergency description, it does not go straight to the LLM. It goes to our Security Checkpoint node. This node scrubs any phone numbers or emails using regex, checks for prompt injection attempts, and writes a structured JSON audit log. 
If the request is safe, the graph routes it to the Check Special Needs node. This is a Human-in-the-Loop step using ADK's RequestInput. It checks if the user mentioned children, pets, or medical issues, and if not, pauses to ask them.
Once the details are collected, the Orchestrator agent takes over. It uses the Agent-as-a-Tool pattern to delegate work to two specialized sub-agents: the Safety Guide agent, which handles survival instructions, and the Resource Locator agent, which finds open shelters and hospitals.
Both sub-agents are wired to our Model Context Protocol, or MCP, server to fetch live alerts, checklists, shelter lists, and hospital status.
Finally, the results are compiled by the Aggregator node, which appends an emergency disclaimer.
